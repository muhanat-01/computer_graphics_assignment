## Project Type and Design Intent
This project is a **residential apartment building**, not an office-rental development. The model was produced as a 14-storey building in Autodesk Revit and documented as a coordinated drawing package covering architecture, electrical services, plumbing services, and network services.

The primary design intent was to deliver:
- A readable and dimensioned apartment floor layout.
- Service drawings that are technically separated and clearly interpreted.
- A complete evidence set with legends, symbols, and labelled outputs aligned to CAT marking requirements.

## Software Decision
The original brief proposed AutoCAD or Archicad. The lecturer later allowed alternative software where system performance was a constraint. Based on that direction, all modelling and drawing work for this submission was completed in **Autodesk Revit**.

## Architectural Documentation (Section a - 10 Marks)

### 1. Floor Strategy and Level Logic
The project uses a **typical upper-floor plan strategy**. One detailed apartment floor was developed and reused on the other upper floors.

Design rule applied:
- The **ground floor** and **rooftop** are intentionally different from the typical upper floor.
- Ground floor differs because of ground-level access conditions (for example, main entrance treatment).
- Rooftop differs because balconies are not part of the roof-level layout.

This approach follows the assignment guidance that one typical floor can represent repeated floors when properly documented.

### 2. Apartment-Oriented Spatial Configuration
The architectural plan was configured for residential use and includes core apartment spaces and circulation. Vertical transport continuity was maintained with staircase and lift coordination through the full height of the building.

What is demonstrated:
- Residential floor planning consistency across repeated levels.
- Practical room-to-circulation relationships for apartment living.
- Vertical continuity of stair and lift cores in both plan and 3D/elevation views.

### 3. Dimensions and Drawing Clarity
Plot/floor geometry and dimensional information are presented directly inside the plan sheets and screenshots. The model output is annotated for technical reading and review.

## Electrical Design Documentation (Section b - 6 Marks)

### 1. Electrical Distribution Intent
Electrical planning was developed as a separate service discipline. Distribution starts from a switchboard located around the staircase zone on the first floor, then feeds apartment-level distribution and endpoints.

### 2. Placement Logic Implemented
Per apartment unit, the following placement rules were applied:
- Kitchen: 1 cooker switch + 1 double socket.
- Living room: 4 double sockets + 1 floor lamp.
- Bedroom: 2 double sockets in each bedroom.
- Balcony: 1 double socket.
- Each house/apartment: 1 distribution board.

### 3. Symbols and Elements Used
- Switch board
- Distribution board
- Double sockets
- Switches
- Cooker socket
- Floor lamps

### 4. Technical Demonstration
The electrical plan demonstrates:
- Functional outlet density by room use.
- Clear service hierarchy from board-level to room-level points.
- Proper symbol-based communication through plan + legend presentation.

## Network Design Documentation (Section d - 5 Marks)

### 1. Network Distribution Intent
A dedicated network plan was developed in the same coordinated model environment to avoid disconnected drafting and to preserve floor-service alignment.

### 2. Placement and Routing Rules Implemented
- LAN/Switch located at the side of the staircase.
- 4 Wi-Fi access points on the floor:
  - 1 around the staircase area.
  - Remaining APs distributed around room-side zones for coverage.
- Data routes represented by **green lines**.

### 3. Network Elements Used
- LAN/Switch
- Wi-Fi access point
- Data routes (green line convention)

### 4. Technical Demonstration
The network plan demonstrates:
- Backbone/distribution concept from central equipment zone.
